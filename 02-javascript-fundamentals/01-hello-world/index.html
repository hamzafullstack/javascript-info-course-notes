JavaScript programs can be inserted almost anywhere into an HTML document using the script tag. The script tag contains JavaScript code which is automatically executed when the browser processes the tag.

Modern markup

The old HTML standard, HTML4, required a script to have a type. Usually it was type="text/javascript". Its not required anymore. Also, the modern HTML standard totally changed the meaning of this attribute. Now, it can be used for JavaScript modules.

External scripts

If we have a lot of JavaScript code, we can put it into a separate file. Script files are attached to HTML with the src attribute:

here is script js file with src

script src="helloscript.js"script

The browser loads the script from the specified URL and executes it. This is useful for keeping HTML files clean and separating concerns.

We can give a full URL as well. and we can attach several scripts, use multiple tags

Please Note

a rule, only the simplest scripts are put into HTML. More complex ones reside in separate files. The benefit of a separate file is that the browser will download it and store it in its cache. Other pages that reference the same script will take it from the cache instead of downloading it, so the file is actually downloaded only once. That reduces traffic and makes pages faster.

Mini Q&A

  1. Do we need type="text/javascript" today? No.
  2. Can one script tag have both src and inline code? No; inline is ignored if src exists
  3. Why prefer external scripts? Caching + reuse across pages.
  4. When does an inline script execute? Exactly when the parser reaches it.
  5. Are language and HTML comment hacks inside script still needed? No; obsolete.
