Lesson: Variables

Core Concepts

JavaScript variables act as named storage for data—think of them like labeled boxes where values are kept and can be changed or copied.

Declaration & Assignment

  • Use let to declare a variable:
    • let message;
  • Assign a value using =, then access using the variable name:
    • message = 'Hello'; → alert(message);
  • You can combine declaration and assignment:
    • let message = 'Hello!';

Style & Readability

  • Avoid multiple declarations in one line (e.g., let a=1, b=2). Instead:
    • let a = 1;
let b = 2;
  • Comma-first style exists but is not recommended.

The Old Way: var

  • var is old-school and block-insensitive—avoid it in modern scripts.

Real-World Analogy

Variables are like boxes with labels—you can put in new values or copy contents to another box.

Avoiding Mistakes

  • Declaring the same variable twice with let causes an error.
  • Choosing good variable names matters:
    • Use letters, digits, $, _.
    • Must not start with a digit.
    • Use camelCase for multi-word names.
    • It's case-sensitive (age ≠ Age).
    • Don't use reserved words like let, class, etc.
