Code structure

Clear & Concise Lesson: Code Structure from javascript.info

Statements

  • Basic units of JavaScript code (e.g., alert('Hello'))..
  • Can be chained with semicolons or placed on separate lines for clarity.

Semicolons & Automatic Insertion

  • Semicolons can often be omitted when there is a newline.
  • But not always. Newlines dont guarantee correct separation
  • Better practice: always use semicolons to avoid subtle bugs

Comments

  • Used to add explanations or notes within the code.
  • Single-line comments: // This is a comment
  • Multi-line comments: /* This is a multi-line comment */
  • Used to explain code, improve readability, and document functionality.
  • Good for documenting, debugging, or temporarily disabling code.
